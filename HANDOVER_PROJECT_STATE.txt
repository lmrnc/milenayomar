GOAL
Build and polish a wedding website for Milena and Omar, matching the printed invitation design direction:
- Artisan white paper feel.
- Fine red botanical pattern as the background.
- Elegant classic typography.
- Functional RSVP form connected to Google Apps Script / Google Sheets.

CURRENT DESIGN DIRECTION
- The red botanical pattern from assets/rojo.webp is used as the background across all sections.
- The green pattern was intentionally removed.
- The hero/inicio section has:
  - Red pattern background with a soft white overlay.
  - No white blurred panel behind the names.
  - Hero text in near-black (#111).
  - Names set in Cormorant Garamond italic for a more classic cursive feel.
- Content sections use white/paper cards for readability on top of the patterned background.
- The RSVP form uses blue ink-like typography and white card surfaces.

- Acompanantes
- Estado

- Notas
- Source

Backend behavior:
- Append-only.
- Does not update or validate against the official guest list.
- New rows use Estado = pendiente and Source = web.
- This supports manual review/validation by the couple.

CURRENT RSVP UX
Initial view:
- Shows only Nombre y apellidos, Email, and Continuar.

After Continuar:
- Shows Datos de asistencia.
- User can select attendance for the main guest.
- User can enter intolerancias and comentarios.
- User can add companions.
- Submit sends native POST to Apps Script and then shows success message.

Important implementation detail:
- Do not replace the native form POST with fetch unless CORS and Apps Script redirects are handled properly.
- The native form + hidden iframe approach is intentional because Apps Script redirects can break fetch/no-cors UX.
